# Preprocess KV Vector Direction

目标：把不同 top-k preprocess KV 看成高维向量，检查随着 top-k 增加时 KV 是否沿相近方向变化。

- cache_root: `/raid/home/hming/fusionrag-pca-topk-cache-5/data`
- dataset: `musique-pca-subset`
- model: `Qwen2.5-7B-Instruct`
- common example/chunk pairs: 121
- metric: cosine between delta vectors, e.g. `(KV_top3 - KV_top1)` vs `(KV_top10 - KV_top1)`.

## 初步解释

- 共享起点的 delta cosine，例如 `1->3` vs `1->10`、`1->5` vs `1->10`，用于判断 top-k 增加是否大体朝同一个目标方向偏移。
- 相邻增量段 cosine，例如 `1->3` vs `3->5`、`3->5` vs `5->10`，用于判断每次新增文档是否沿同一条直线继续推进。
- 如果共享起点 cosine 高、但相邻增量 cosine 低，说明整体有稳定目标方向，但路径不是简单线性叠加；新增文档会引入新的局部方向。

## 当前结论

1. 随着 top-k 增加，KV 的确存在一个中等偏强的整体偏移方向。
   - value: `1->3` vs `1->10` 的 mean cosine 为 `0.7022`，`1->5` vs `1->10` 为 `0.7907`。
   - key: `1->3` vs `1->10` 为 `0.7229`，`1->5` vs `1->10` 为 `0.8218`。
   - 这说明 top-k 增加不是随机扰动，而是大体把 KV 往 top10/fuller-prefix 表示的方向拉。

2. 但这个过程不是简单的一条直线。
   - value: `1->3` vs `3->5` 的 mean cosine 为 `-0.2307`，`3->5` vs `5->10` 为 `-0.2460`。
   - key 也有类似现象。
   - 这说明每新增一批 prefix 文档，会引入新的局部修正方向；最终结果有一致目标方向，但中间路径不是线性叠加。
